### Pain points - User POV
- Phone app to check in
    - QR codes on desks to check-in when sat down
- Chat bot to book
    - Talk to a chat book and describe what you need
    - Desk bookings
    - Room bookings
- Clashing bookings
    - If 2 users are trying to book a desk/room via different methods, there needs to be a way to ensure there are no double bookings
    - If we have a mobile app and a desktop version, there needs to be way to prevent clashes.
    - Single source of truth
- Role-based colouring for identification of teams
    - Desks are coloured by what
- Favouriting of others
- Live counts of availabilities
    - Percentage or number of capacity
    - Show days which are the quietest
- Queue/waitlist of desks
    - Notified when desks are available
- Team-based alerting of desks
    - Team blocks off area for team
    - Invite to prompt user to make a booking
    - Release after period to make available
- Be able to edit your booking

### Pain points - Admin POV
- Report building
    - Data graphing utilisation
- Non arrivals spoiling data
- Non check in's spoiling data
    - If users don't check in, the desk is still booked
